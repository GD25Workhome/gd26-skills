# ai_docs Markdown 文件命名（YYMMDDNN）

## 何时启用

- 需要 **新建并保存** `.md` 文件，且用户 **未给出完整文件名**。
- 目标路径为项目约定的 **`ai_docs/`**（或用户另行指定的目录，规则相同）。
- 其它 skill（如 **analyze-code-entrypoint**）要求产出设计/说明文档并落盘时，**在写入文件前**执行本技能。

## 文件名格式

`YYMMDDNN-文档描述.md`

| 段 | 含义 | 规则 |
|----|------|------|
| **YY** | 年份后两位 | 如 2026 → `26` |
| **MM** | 月份 | 01–12，不足两位前补 0 |
| **DD** | 日 | 01–31，不足两位前补 0 |
| **NN** | 当天序号 | 两位数字 `01`–`99`，**同一 YYMMDD 下递增** |
| **文档描述** | 标题或短描述 | 建议简短、可读，避免与序号规则冲突 |

完整前缀为 **8 个数字** `YYMMDDNN`，其后接 **一个连字符 `-`**，再接描述与扩展名。

## 必须执行的步骤（禁止跳步）

1. **取当日日期**：使用**当前实际日期**（以对话或系统日期为准），格式化为 **YYMMDD**（6 位）。
2. **列出目标目录**：使用工具 **列出** 目标目录内容（默认仓库根目录下的 `ai_docs/`；若用户指定了别的目录，则用该目录）。
3. **筛选当天文件**：在列表中找出所有 **`YYMMDD*.md`** 且前缀 **恰好以当日 6 位日期开头** 的文件（即文件名形如 `YYMMDD` + 两位序号 + `-`…）。
4. **提取序号 NN**：对每个匹配文件，取文件名 **第 7、8 位**（紧跟在 `YYMMDD` 后的两位数字）作为 `NN`。
5. **计算新序号**：若当天无任何匹配文件，则 **`NN = 01`**；否则 **`NN = max(已有 NN) + 1`**，并格式化为两位（如 `9 → 10` 写作 `10`，`01` 保持两位）。
6. **拼出文件名**：`YYMMDD` + `NN` + `-` + `文档描述.md`，确认不与已有文件重名后写入。

## 工具与实现说明

- **必须用当前文件系统结果**，禁止凭记忆猜测目录里已有文件。
- 若环境无 `list_dir`，可用 **递归/非递归的目录列举** 或 **`glob_file_search`**（例如 `ai_docs/YYMMDD*.md`）达到同等效果。
- 若仓库内 `ai_docs` 不存在，**先创建目录** 再写入；当天首文件仍为 `YYMMDD01-….md`。

## 禁止行为

- 不扫描目录就定序号。
- 使用「大概今天第几个」而不根据目录内文件计算最大 NN。
- 与其它日期前缀的文件混算序号（只认**当天 YYMMDD**）。
- 生成与已存在文件完全相同的文件名。

## 示例

假定今天为 **2026 年 4 月 3 日**，则 `YYMMDD = 260403`。

- 目录中已有：`26040301-接口说明.md`，则新文件应为 **`26040302-新文档标题.md`**。
- 目录中当天无 `260403*.md`，则新文件应为 **`26040301-新文档标题.md`**。
