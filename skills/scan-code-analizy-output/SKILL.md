# 扫描输出影响技能

## 适用场景

- 用户要求“扫描某个入口的所有输出”
- 用户要求“输出驱动分析/不要漏细节”
- 需要同时梳理：返回结果、数据库变更、第三方接口或组件调用

## “输出”的定义

- **返回结果**：HTTP/RPC/方法返回对象，以及异常分支可观察返回
- **数据库变更**：会改变持久化状态的写操作（INSERT/UPDATE/DELETE/逻辑删除/批量写入）
- **第三方接口或组件调用**：跨边界调用，包括 HTTP、RPC、MQ、缓存、搜索、对象存储、外部 SDK、跨模块网关适配层调用
- **排除项**：普通日志输出（`log.info/warn/error/debug`）不计入输出
- **不可以忽略**：如果调用了异步的接口，异步接口中的逻辑必须也扫描，检查其是否有满足上面对“输出”定义的字段，然后将其囊空在输出的分析字段范围内

## 执行流程（必须按顺序）

1. 先扫描整个代码链路，明确所有代码分支
  - 要求是你所有可以触达的代码
  - 不可以忽略异步线程运行的代码分支，其内部涉及的输出必须囊括在文档中
2. 分析代码链路中的所有输出
  - 输出抽取：按“返回结果/数据库变更/第三方调用”三类收集字段级信息。
  - 排除噪音：明确剔除普通日志输出，避免把日志当业务输出。

## 输出模板（固定）

按以下结构输出，不能省略章节：

```markdown
## 1) 入口信息
- 入口方法：
- 路由/触发方式：
- 扫描范围：

## 2) 返回结果输出
- 成功返回：
  - 对象：
  - 字段：
- 失败返回：
  - 异常/错误码：
  - 触发条件：

## 3) 数据库变更输出
- 表/实体：
  - 变更类型：
  - 字段：
  - 简要概述：

## 4) 第三方接口/组件调用输出
- 组件/客户端：
  - 调用动作：
  - 请求关键字段：
  - 返回关键字段：
  - 简要概述：

## 5) 排除说明
- 已排除普通日志输出（log.*）及无副作用中间变量。
```

## 质量要求

- 禁止只给结论，必须给字段级清单。
- 禁止忽略异步分支。
- 禁止把普通日志计入输出。
- 若未发现数据库变更或第三方调用，必须明确写“未发现”。

## 自检清单
- [ ] 异步接口中的输出项目是否存在
- [ ] 是否忽略了异步接口中的输出项目

# 落盘规则

- 默认将结果写成 Markdown 落盘至仓库 `ai_docs/`。
- 文件名遵循 `ai-docs-md-naming`（`YYMMDDNN-描述.md`）。
- 若用户要求仅对话输出、禁止生成文件或已指定路径，则以用户要求为准。
