# 扫描输出影响技能

## 适用场景

- 用户要求“扫描某个入口的所有输出”
- 用户要求“输出驱动分析/不要漏细节”
- 需要同时梳理：返回结果、数据库变更、第三方接口或组件调用

## 落盘规则

- 默认将结果写成 Markdown 落盘至仓库 `ai_docs/`。
- 文件名遵循 `ai-docs-md-naming`（`YYMMDDNN-描述.md`）。
- 若用户要求仅对话输出、禁止生成文件或已指定路径，则以用户要求为准。

## 输出定义

- **返回结果**：HTTP/RPC/方法返回对象，以及异常分支可观察返回
- **数据库变更**：会改变持久化状态的写操作（INSERT/UPDATE/DELETE/逻辑删除/批量写入）
- **第三方接口或组件调用**：跨边界调用，包括 HTTP、RPC、MQ、缓存、搜索、对象存储、外部 SDK、跨模块网关适配层调用
- **排除项**：普通日志输出（`log.info/warn/error/debug`）不计入输出

## 执行流程（必须按顺序）

1. 锁定入口：确认路由/注解/方法签名与真实实现。
2. 主链追踪：沿调用链向下，定位返回对象构造点、写库点、外部调用点。
3. 异步追踪：继续跟踪 `CompletableFuture`、线程池、事件、MQ、定时任务等分支。
4. 输出抽取：按“返回结果/数据库变更/第三方调用”三类收集字段级信息。
5. 排除噪音：明确剔除普通日志输出，避免把日志当业务输出。
6. 结果校验：若某类未命中，必须显式写“未发现”。

## 识别规则

### 返回结果

- 识别成功返回对象字段（含包装体与 data 字段）
- 识别异常分支（异常类型、错误码/错误文案、触发条件）

### 数据库变更

- 识别操作：`insert/save/saveBatch/update/updateById/updateBatchById/delete/remove/removeByIds`
- 每个变更点必须包含：
  - 表名或实体名
  - 变更类型（INSERT/UPDATE/DELETE）
  - 字段清单（实际 set/赋值字段）
  - 简要概述（1 句话说明业务意义）

### 第三方接口或组件调用

- 重点识别：
  - HTTP 客户端（Feign/RestTemplate/WebClient/OkHttp/HttpClient）
  - RPC 客户端（Dubbo/gRPC/自研 RPC）
  - 中间件（Redis/MQ/ES/对象存储）
  - 外部平台 SDK（短信/邮件/支付/地图等）
  - 跨模块网关或适配组件
- 每个调用点必须包含：
  - 组件/客户端名称
  - 调用动作（如 POST、publish、set）
  - 请求关键字段
  - 响应/回执关键字段（若可见）
  - 简要概述

## 输出模板（固定）

按以下结构输出，不能省略章节：

```markdown
## 1) 入口信息
- 入口方法：
- 路由/触发方式：
- 扫描范围：

## 2) 返回结果输出
- 成功返回：
  - 对象：
  - 字段：
- 失败返回：
  - 异常/错误码：
  - 触发条件：

## 3) 数据库变更输出
- 表/实体：
  - 变更类型：
  - 字段：
  - 简要概述：

## 4) 第三方接口/组件调用输出
- 组件/客户端：
  - 调用动作：
  - 请求关键字段：
  - 返回关键字段：
  - 简要概述：

## 5) 排除说明
- 已排除普通日志输出（log.*）及无副作用中间变量。
```

## 质量要求

- 禁止只给结论，必须给字段级清单。
- 禁止忽略异步分支。
- 禁止把普通日志计入输出。
- 若未发现数据库变更或第三方调用，必须明确写“未发现”。
