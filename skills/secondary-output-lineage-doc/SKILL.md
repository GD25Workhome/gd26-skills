# 二次输出链路文档技能

## 适用场景

- 用户已提供一份“输出扫描”基线文档（例如 `cursor_docs/26041301-*.md`）。
- 用户再提供一个代码入口（Controller/Service 方法）要求继续深挖。
- 用户要求：
  - 字段级来源拆解（直接输入/查询/计算/覆盖）；
  - 复杂字段补充；
  - 按复杂度标记重点关注项；
  - 保持文档整体结构与可读性。

## 落盘规则

- 默认将结果写成 Markdown 落盘至仓库 `ai_docs/`。
- 文件名遵循 `ai-docs-md-naming`（`YYMMDDNN-描述.md`）。
- 若用户要求仅对话输出、禁止生成文件或已指定路径，则以用户要求为准。

## 强制输入（缺一不可）

1. **基线文档路径**：必须是本仓库内已存在的 markdown 文档。
2. **代码入口定位**：必须给出文件路径 + 方法（或行号片段）。

若缺任一输入，先向用户追问，不得直接输出完整分析结论。

## 强制门禁（必须按顺序）

1. **先读基线文档**  
   - 抽取已覆盖字段、已说明链路、已标记风险。
2. **再读入口代码与下游实现**  
   - 必须覆盖同步链路 + 异步链路。
3. **做“字段覆盖审计”**  
   - 对每个输出字段判断：`已充分展开 / 部分充分 / 未覆盖`。
4. **只扩写“复杂且未充分展开”的字段**  
   - 避免重复重写已充分内容。
5. **写回统一结构文档**  
   - 新增或更新章节时保持“先总后分”。

## 分析口径（字段级）

对每个字段至少给出以下 6 项中的 4 项（复杂字段尽量给全）：

1. 字段落点（返回体/表名/实体字段）
2. 输入来源（请求、登录态、配置、数据库、外部系统）
3. 生成逻辑（拼接、映射、聚合、排序、阈值、时间线）
4. 覆盖顺序（首次赋值、二次修正、最终主导来源）
5. 默认值/兜底/异常分支
6. 风险与验证建议

## 复杂度评级标准

- `L1-低`：直接透传或简单常量赋值。
- `L2-中`：单层计算或单次映射，无覆盖。
- `L3-高`：多输入融合、条件分支明显，或存在一次以上覆盖。
- `L4-极高`：跨方法/跨阶段策略链路，存在时间线、排序选取、口径不对称、反向回写等。

## 输出模板（固定）

```markdown
## 1) 入口与基线
- 入口：
- 基线文档：
- 本次补充目标：

## 2) 基线覆盖审计
| 字段/字段组 | 基线覆盖度 | 本次是否扩写 | 原因 |
|---|---|---|---|

## 3) 复杂字段补充（仅补充项）
### 3.x [字段或字段组]
- 落点：
- 输入来源：
- 生成逻辑：
- 覆盖顺序：
- 兜底/异常：
- 复杂度评级：
- 风险：

## 4) 复杂度总览与关注优先级
| 字段 | 复杂度 | 展开充分性 | 建议优先级 |
|---|---|---|---|

## 5) 待确认项（Unknown）
- 字段：
- 缺失原因：
- 建议补扫位置：
```

## 文档结构要求

- 先总览后细节，禁止直接堆字段。
- 章节编号连续，新增章节要同步调整后续编号。
- 同一术语保持一致：
  - “输入来源”
  - “覆盖顺序”
  - “复杂度评级”
  - “展开充分性”

## 质量检查清单

- [ ] 已读取“基线文档 + 入口代码”
- [ ] 已覆盖同步与异步链路
- [ ] 已输出“基线覆盖审计表”
- [ ] 仅扩写复杂且未充分字段，未重复冗余重写
- [ ] 每个 L4 字段都给出具体复杂原因
- [ ] 明确列出 Unknown 与下一步补扫点

## 禁止事项

- 禁止脱离基线文档直接重写全篇。
- 禁止仅给结论不列字段。
- 禁止把普通日志当业务输出字段。
- 禁止把“可能”当“已证实”，未闭环必须标记 `Unknown`。
